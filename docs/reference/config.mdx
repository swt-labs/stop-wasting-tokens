# Configuration reference

`config.json` lives at `.swt-planning/config.json` and is read on every `swt` invocation. All keys are optional — defaults from `@swt-labs/core/config/defaults.json` apply where missing.

### `active_profile`

- **Type:** `string`
- **Default:** `"default"`
- **When to override:** _(no override guidance documented)_

### `agent_max_turns`

- **Type:** `Record<string, number>`
- **Default:** `{"scout":15,"qa":25,"architect":30,"debugger":80,"lead":50,"dev":75}`
- **When to override:** Bump `qa` if your QA agent regularly hits the cap on dense phases. The shipped defaults (15/25/30/80/50/75 for scout/qa/architect/debugger/lead/dev) reflect the v1.0 ship's empirical floor.

### `auto_push`

- **Type:** `"never" | "after_phase" | "always"`
- **Default:** `"never"`
- **When to override:** `"after_phase"` for solo projects with a remote; `"always"` only for ephemeral branches.

### `auto_uat`

- **Type:** `boolean`
- **Default:** `false`
- **When to override:** `false` to require explicit `swt vibe --verify` invocations. The default `true` is right for solo / pure-vibe workflows; teams may prefer explicit verification gates.

### `autonomy`

- **Type:** `"cautious" | "standard" | "confident" | "pure-vibe"`
- **Default:** `"standard"`
- **When to override:** `pure-vibe` for solo mature projects, `cautious` for regulated work. See [autonomy levels](/concepts/autonomy-levels).

### `hooks`

- **Type:** `object (optional)`
- **When to override:** Wire shell scripts to SDLC lifecycle events. Each event accepts an optional `{ script_path: "..." }` block. The script runs synchronously when the event fires; non-zero exit aborts the lifecycle action (e.g., `pre_archive` exiting 1 aborts the archive). See `templates/hooks/*.sample.sh` for shell script templates.
