### `swt config`

Read or update SWT configuration.

**Usage:** `swt config [show|get <key>|set <key> <value>]`

**Notes:** Read or update SWT configuration. `swt config show` prints the resolved config; `swt config get <key>` returns a single value; `swt config set <key> <value>` mutates `.swt-planning/config.json`. Validation runs through `ConfigSchema`, so invalid values are rejected with a clear error.

### `swt update`

Check the npm registry for a newer published version.

**Usage:** `swt update [--json] [--strict] [--registry=<url>] [--no-cache]`

**Notes:** Checks the npm registry for a newer published version. With `config.marketplace.endpoint` set, also queries the Codex Plugin Marketplace and flags any version divergence between the two sources. `--no-marketplace` skips the marketplace path even when configured.

### `swt vibe`

Detect project state and route into the right SDLC mode.

**Usage:** `swt vibe [N] [--effort=level] [--yolo] [--skip-qa] [--plan=NN]`

**Notes:** The `vibe` command is the SDLC entry point. With no arguments, it routes based on the current planning state (init-redirect → bootstrap → scope → plan → execute → verify → archive). Pass a phase number (`swt vibe 03`) or a mode flag (`swt vibe --plan 03`) to target a specific stage.

### `swt watch`

Open an Ink TUI dashboard scoped to the active milestone.

**Notes:** Opens an Ink TUI dashboard scoped to the active milestone. Updates within 1 second on `.swt-planning/` file-system changes. Read-only — does not modify any files. Ctrl+C exits cleanly. See [F4 in the v1.5 roadmap](/roadmap/v1.5#f4-ink-tui-watch).
