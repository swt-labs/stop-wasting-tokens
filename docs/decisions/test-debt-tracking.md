# v3 test-debt tracking

Authoritative inventory of every test currently skipped or deleted as part of the v2.3.5 → v3 transition. Created at M1 Plan 01-03 PR-11 Task A per TDD2 §13.1.4 checklist item 5. Each row is either:

- **Deleted** — the test referenced code or artefacts that v3 removed (driver packages, the `.codex-plugin/` directory, v1-era launch announcements). The test file is gone from `v3-foundation`.
- **Skipped** — the test is real-but-blocked behind the umbrella issue [#32](https://github.com/swt-labs/stop-wasting-tokens/issues/32). The test file uses `describe.skip(...)` at the top level so the test bodies stay intact for restoration; a header comment + `// TODO(v3-debt)` annotation points at #32.

When the umbrella issue closes:

1. Each cluster's rows below are marked Resolved (or split into per-cluster issues if the work is scoped that way).
2. The `describe.skip(...)` calls are reverted to `describe(...)` and the test bodies updated to the v3 contract.
3. This file is updated to record the resolution PR.

## What the next maintainer needs to know

1. **Most skips are cluster-level.** When you unskip a `describe.skip` block, run the file individually first to see what the current contracts look like before you assume the test bodies still apply.

2. **`packages/verification/test/guards.test.ts` is a security issue.** Treat it as a v3 hotfix candidate, not a deferred-test-debt item. The describe.skip wrap is a stop-the-bleeding measure so CI's required Test step doesn't block other M1+ work; the fix should land within the next 1–2 PRs.

3. **The umbrella issue [#32](https://github.com/swt-labs/stop-wasting-tokens/issues/32)** is the GitHub-side tracking anchor. Each entry above maps to a section of that issue.

4. **CI behavior after PR-11 Task A:** the `Test` step in `.github/workflows/ci.yml` is now required (no `continue-on-error: true`). Any new test failure blocks merge; old failures stay green because their files are skipped at the describe level.

5. **Restoring a skipped test:** replace `describe.skip(` with `describe(` at the top-level of the file + delete the 3-line `// TODO(v3-debt)` header annotation. Update this file's row to "Resolved (PR-XX, YYYY-MM-DD)" + the umbrella issue's checkbox.
