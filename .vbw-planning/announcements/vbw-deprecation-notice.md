> **VBW is now in maintenance mode. v1.0.97-final is the last release.**
>
> The methodology lives on as **stop-wasting-tokens (SWT)** — a portable npm CLI for Codex, with Claude Code and Ollama drivers landing in v1.5. Migration is `mv .vbw-planning .swt-planning`.
>
> - **SWT docs:** https://docs.stopwastingtokens.dev
> - **Migration guide:** https://docs.stopwastingtokens.dev/migration/from-vbw
> - **Repo:** https://github.com/swt-labs/stop-wasting-tokens
> - **npm:** `npm install -g @swt-labs/cli`
>
> This VBW repo will remain accessible as a read-only reference at the `v1.0.97-final` tag. Issues and PRs will not be actively triaged. The Contributor Covenant 2.1 community standard remains in effect for archived discussions.
>
> Thank you to every VBW user who shaped the methodology. The eleven lifecycle states, the 7-point audit, the QA + UAT remediation pipelines — all of it ships intact in SWT, with frontmatter shapes wire-compatible (Zod transforms accept both VBW and SWT forms).
>
> If you're running a project on VBW today, you do not need to migrate immediately. v1.0.97-final stays installable. But new feature work, bug fixes, and the multi-driver future all live in SWT.
>
> — SWT maintainers
