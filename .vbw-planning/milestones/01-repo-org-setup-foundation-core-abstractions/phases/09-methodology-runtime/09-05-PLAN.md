# Phase 9 / Plan 05: QA + UAT verification pipelines

## Why fifth

PLAN 04 ships Plan + Execute — phase code lands on disk. PLAN 05 ships the verification side: QA agent integration, VERIFICATION.md / UAT.md writers, known-issues lifecycle, and the qa/verify/re-verify handlers that drive `swt vibe` past the verification stages of the routing table. After this lands, the runtime can execute a phase end-to-end: bootstrap → scope → plan-and-execute → qa → verify (modulo the inline UAT checkpoint UX, which is PLAN 06's job).

## Tasks

### T1 — artifacts/qa/verification.ts
`writeVerification({phaseDir, payload, tier, verifiedAtCommit, completed})` formats
the existing `QaVerificationPayload` (phase, plans_verified, result, checks[], pre_existing_issues[])
plus tier and verified_at_commit into a YAML frontmatter + markdown body, written via
`writeAtomically` to `<phaseDir>/<NN>-VERIFICATION.md`. `readVerification(phaseDir, phase)`
parses back via `parseFrontmatter` and validates with `QaVerificationPayloadSchema`.

### T2 — artifacts/qa/uat.ts
`writeUat({phaseDir, phase, planCount, status, totalTests, passed, skipped, issues, completed})`
emits `<phaseDir>/<NN>-UAT.md` with frontmatter (`phase`, `plan_count`, `status`,
`started`, `completed`, `total_tests`, `passed`, `skipped`, `issues`) and an issues body
section. Schema-validate via Zod (UatPayloadSchema lives next to the writer).

### T3 — artifacts/qa/known-issues.ts
`KnownIssueSchema` carries `{id, severity: 'critical'|'major'|'minor'|'cosmetic',
summary, opened_at, status: 'open'|'resolved'|'deferred', resolution_round?}`.
`readKnownIssues(phaseDir)` returns `[]` when file is missing; `writeKnownIssues`
uses `writeAtomically` with stable JSON ordering. Add idempotent helpers `addIssue`,
`resolveIssue`, `deferIssue`.

### T4 — artifacts/qa/remediation-state.ts
`getOrInitRemediationState(phaseDir, severity)` reads `<phaseDir>/.uat-remediation-stage`
(JSON: `{round, layout, severity, started, last_stage}`). Initializes round=01, layout='round-dir'
when missing. `advanceRemediationRound(phaseDir)` increments `round` and writes back.

### T5 — methodology/src/qa/freshness.ts
`checkQaFreshness({phaseDir, phase, cwd})` reads `<phaseDir>/<NN>-VERIFICATION.md`,
extracts `verified_at_commit`, runs `git rev-parse HEAD` via execFile under `cwd`,
and returns `passed` (PASS + commit match), `remediated` (PASS + ahead of verified
commit but verifying the same files), `pending` (no VERIFICATION.md or malformed),
or `failed` (result=fail). Mirrors phase-detect.sh's qa_status mapping.

### T6 — methodology/src/qa/checklist.ts
`synthesizeUatChecklist({phaseDir, plans})` reads each PLAN.md frontmatter,
extracts `must_haves[]` and `acceptance_criteria`, and builds a `UatChecklist`
with one test row per must-have (status='pending' until inline checkpoints in PLAN 06
populate them). For PLAN 05 the synthesized checklist is written to UAT.md with
status='deferred' on each row (PASS aggregate when all rows are deferred — matches
the mechanical UAT pass shape used today).

### T7 — handlers/qa.ts
`qaHandler` reads `<phaseDir>/<NN>-NN-SUMMARY.md`, spawns `qa` via AgentSpawner,
parses the `qa-verification` handoff via `parseQaHandoff`, writes VERIFICATION.md,
updates known-issues.json. Throws `NotImplementedError` when no spawner is injected
(parity with executeHandler — Codex driver lands later). Registered for both
`kind='qa-remediation'` and triggered inline by verifyHandler when `qa_pending=true`.

### T8 — handlers/verify.ts
`verifyHandler` synthesizes the UAT checklist from plan must-haves, writes UAT.md
with `status='deferred'` rows (PASS aggregate), updates known-issues.json with any
issues the user pre-recorded. The inline checkpoint UX (per-row PASS/FAIL prompts)
is PLAN 06's responsibility; this handler ships the artifact-write path.

### T9 — handlers/re-verify.ts
`reVerifyHandler` runs the prepare-reverification flow: reads existing `<NN>-UAT.md`,
moves it into `<phaseDir>/remediation/uat/round-{RR}/R{RR}-UAT.md`, bumps the
remediation-state round, and returns HandlerResult so the next /vbw:vibe pass
re-enters Verify against the cleared slate.

### T10 — Wire into vibe registry
Update `packages/cli/src/commands/vibe.ts` to register `qaHandler`, `verifyHandler`,
and `reVerifyHandler` alongside the existing handlers via `buildVibeRegistry`.
