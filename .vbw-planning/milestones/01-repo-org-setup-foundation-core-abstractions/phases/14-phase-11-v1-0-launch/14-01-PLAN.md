# Phase 14 / Plan 01: RELEASE-NOTES-v1.0 + CHANGELOG + launch blog post

## Why first

The release notes anchor the rest of Phase 14. Security review (PLAN 14-02) cross-references them. VBW deprecation notice (PLAN 14-03) summarizes them. The blog post + CHANGELOG are the public-facing artifacts that need to be ready before the user posts the announcements (Phase 13's `.vbw-planning/announcements/` templates).

## Layout

```
RELEASE-NOTES-v1.0.md            # NEW — comprehensive launch summary
CHANGELOG.md                     # NEW — Keep-a-Changelog

  v1-0-launch.mdx                # NEW — 800-word launch post
docs/docs.json                   # +Blog navigation group
docs/test/release-notes.test.ts  # NEW — version + phase coverage assertion
README.md                        # status table refresh + RELEASE-NOTES link
```

## Tasks

### T1 — RELEASE-NOTES-v1.0.md
Author at repo root. Sections:

- **Headline + 30-second pitch.**
- **What's in v1.0** — narrative covering the 13 shipped phases grouped by theme: Foundation (1-2), Abstractions + Drivers (3-4), Methodology + Commands (5-6), Artifacts + Verification (7-8), Runtime + Templates (9-10), Docs + Distribution (11-12), Beta (13). Each theme is one paragraph + a 2-3 bullet "what shipped" list.
- **VBW compatibility.** A paragraph on the migration story: `mv .vbw-planning .swt-planning`, frontmatter shapes accept both forms via Zod transforms.
- **Install + quickstart.**
- **What's intentionally not in v1.0** — real Codex AgentSpawner wiring (v1.5), Claude Code + Ollama drivers (v1.5), Ink TUI (v1.5), live deployment to docs.stopwastingtokens.dev (gated on user-side hosting setup).
- **Acknowledgments** — VBW community (Hesreallyhim and contributors), Anthropic + OpenAI for the LLM substrate, beta testers.
- **Links** — docs, repo, npm, marketplace listing (placeholder until accepted).

### T5 — release-notes vitest + README refresh
Author `docs/test/release-notes.test.ts`:

```typescript
import { describe, expect, it } from 'vitest';
import { readFileSync } from 'node:fs';
import { join } from 'node:path';

const REPO_ROOT = join(__dirname, '..', '..');

describe('RELEASE-NOTES-v1.0', () => {
  const notes = readFileSync(join(REPO_ROOT, 'RELEASE-NOTES-v1.0.md'), 'utf8');

  it('references all 13 prior phases', () => {
    const phaseNames = [
      'Repo', 'Foundation', 'Core Abstractions', 'Codex',
      'Methodology Authoring', 'Commands', 'Artifacts', 'Verification',
      'Methodology Runtime', 'Template Fidelity', 'Documentation',
      'Distribution', 'Beta',
    ];
    for (const name of phaseNames) {
      expect(notes, `RELEASE-NOTES missing phase: ${name}`).toContain(name);
    }
  });

  it('references the 4 core abstractions', () => {
    for (const abs of ['HookHost', 'AgentSpawner', 'PermissionGate', 'MemoryStore']) {
      expect(notes, `RELEASE-NOTES missing abstraction: ${abs}`).toContain(abs);
    }
  });

  it('references the 11 lifecycle states (count, not enumerated)', () => {
    expect(notes).toMatch(/11.*lifecycle.*states?/i);
  });
});
```
