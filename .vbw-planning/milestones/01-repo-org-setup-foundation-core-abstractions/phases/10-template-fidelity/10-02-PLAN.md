# Phase 10 / Plan 02: VBW-grade VERIFICATION body sections + RESEARCH schemas

## Why second

PLAN 10-01 covered PLAN.md and SUMMARY.md frontmatter. PLAN 10-02 layers the next two highest-value documents on top: VERIFICATION.md (its multi-table body sections drive the QA + audit gates) and RESEARCH.md (the deterministic input to Lead during planning). Together with PLAN 10-01, the round-trip surface now covers every artifact the orchestrator reads or writes during a normal phase lifecycle.

## Layout

```
packages/artifacts/src/qa/verification.ts        # extend with body parser + multi-section renderer
packages/artifacts/src/qa/uat.ts                 # extend with severity_counts
packages/artifacts/src/schemas/research.ts       # new: ResearchFrontmatterSchema + StandaloneResearchSchema
packages/artifacts/src/schemas/index.ts          # extend the barrel
packages/artifacts/test/qa/verification-body.test.ts
packages/artifacts/test/qa/uat-severity.test.ts
packages/artifacts/test/schemas/research.test.ts
packages/artifacts/test/fixtures/vbw-verification-multi-section.md
```

## Tasks

### T1 — verification body parser/renderer
Extend `verification.ts`:
- `VerificationDocSchema` adds `artifact_checks`, `key_link_checks`, `anti_pattern_checks`, `convention_checks`, `requirement_mapping` arrays + an optional `layout: 'swt' | 'vbw'` (default `'swt'`).
- `parseVerificationBody(body)` reads `## {Section}` headings and, when followed by a `| ID | ... | Status | Evidence |` table, parses rows into the matching typed array. Existing single-table body parsing still wins when only one table is present.
- `renderVerificationBody(doc)` produces the SWT default single-table body when `layout='swt'` and the VBW multi-section layout when `layout='vbw'`.

### T2 — UAT severity counts
- `UatDocSchema` adds `severity_counts: {critical, major, minor, cosmetic}` (computed when not supplied).
- `writeUat` renders a `Severity Mix` line at the top of the Issues body section when `severity_counts` is present.

### T3 — RESEARCH schemas
- `schemas/research.ts` exports `ResearchFrontmatterSchema` (phase, plan?, gathered, sources_consulted, findings_summary, files_referenced, live_validation_required) and `StandaloneResearchFrontmatterSchema` (topic instead of phase/plan).
- `readResearchFrontmatter` / `writeResearchFrontmatter` helpers.

### T4 — Test fixtures
`vbw-verification-multi-section.md`: a verbatim VBW VERIFICATION.md with all six tables.

### T5 — Vitest
- `verification-body.test.ts`: round-trip the multi-section fixture; render SWT default vs VBW multi-section; backwards-compat single-table parse.
- `uat-severity.test.ts`: derive severity_counts from issue_records of mixed severities; round-trip with explicit severity_counts.
- `research.test.ts`: parse + write phase research; parse + write standalone research; reject malformed gathered date.

### T6 — Commit + summary
`feat(artifacts): VBW-grade VERIFICATION body + UAT severity + RESEARCH schemas (Phase 10 / PLAN 02)`.
