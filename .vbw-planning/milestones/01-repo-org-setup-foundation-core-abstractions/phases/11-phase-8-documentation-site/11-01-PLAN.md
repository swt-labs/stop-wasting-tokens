# Phase 11 / Plan 01: Mintlify scaffold + getting-started + concepts

## Why first

The docs site needs a stable scaffold before any reference content lands. Plan 01 establishes the navigation taxonomy, theme, and the two highest-traffic sections (Getting Started + Concepts) so PLAN 11-02 can drop reference/recipes/migration content into a working frame. Mintlify is chosen over Docusaurus because (a) zero-config theming, (b) first-class MDX components for API tables, (c) Anthropic, Resend, Cal.com, and other dev-tool projects use it — pattern-match for SWT's audience.

Author `apps/docs/test/structure.test.ts`:
```typescript
import { describe, expect, it } from 'vitest';
import { readFileSync, existsSync } from 'node:fs';
import { join, dirname } from 'node:path';

const DOCS_ROOT = join(__dirname, '..');
const config = JSON.parse(readFileSync(join(DOCS_ROOT, 'docs.json'), 'utf8'));

describe('docs.json structure', () => {
  it('has the 6 expected top-level navigation groups', () => {
    const groups = config.navigation.map((n: any) => n.group);
    expect(groups).toEqual([
      'Getting Started', 'Concepts', 'Reference',
      'Recipes', 'Migration', 'v1.5 Roadmap',
    ]);
  });

  it('every page reference resolves to an .mdx file', () => {
    for (const group of config.navigation) {
      for (const page of group.pages ?? []) {
        const path = typeof page === 'string'
          ? join(DOCS_ROOT, `${page}.mdx`)
          : null;
        if (path) expect(existsSync(path), `missing: ${page}`).toBe(true);
      }
    }
  });
});
```
Wire into root vitest config so `pnpm test` picks it up.
