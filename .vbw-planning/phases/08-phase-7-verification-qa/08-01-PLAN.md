## Tasks

### T1 — PostToolUse checks
- `summary-frontmatter.ts` validates that a SUMMARY.md has phase/plan/title/status/ac_results/files_modified/commit_hashes/deviations.
- `commit-message.ts` enforces Conventional Commits prefix.
- `plan-frontmatter.ts` validates phase/plan/title/wave/must_haves/acceptance_criteria.

Each helper returns `{ ok: boolean; reasons?: readonly string[] }`.

### T2 — PreToolUse guards
- `bash-guard.ts` parses compound bash commands (split on `;`, `&&`, `||`, `|`), checks each segment against a denylist (`rm -rf /`, `sudo`, `curl | sh`, `:>` to system paths, etc.), returns `HookOutcome` from core.
- `file-guard.ts` checks that a write target is inside an allowed-roots prefix list.
- `secret-scanner.ts` scans content for high-confidence secret patterns (AWS access keys, GitHub tokens, slack tokens, generic 40-character hex tokens) and blocks when matched.

### T3 — Compaction circuit breaker
A `CompactionCircuitBreaker` class with `recordFailure()` and `recordSuccess()`. Trips after 3 consecutive failures, resets on success. `isTripped()` returns boolean.

### T4 — Traceability checker
`checkTraceability({ requirements, roadmap, plans, summaries })` returns a structured report listing:
- requirement IDs not referenced in any plan
- plans whose `requirements` field references a non-existent REQ-ID
- plans without a matching SUMMARY.md
- SUMMARYs that don't satisfy any plan must-have

### T5 — swt qa runner
Pure data-in / data-out: `runQa({ tier, plan, summary, verifications? })` returns the QaVerificationPayload shape from core. Tier resolves which evidence is required (must-have evidence required from `standard` upward; traceability required at `deep`).

### T6 — Tests
- One assertion suite per check / guard
- Circuit breaker — 3 fails → trip, reset on success
- Traceability — unmapped REQ, plan w/o SUMMARY, dangling REQ ref
- Runner — pass / partial / fail outputs at each tier

### T7 — Commit + summary
`feat(verification): PostTool checks, PreTool guards, traceability, circuit breaker, qa runner`.
