<objective>
Add a `swt watch` command that opens an Ink TUI dashboard scoped to the active milestone. The dashboard re-renders on `.swt-planning/` file-system changes (chokidar-driven), closes cleanly on Ctrl+C, and works cross-platform.
</objective>
<context>
**Decision resolved 2026-05-07 (user via AskUserQuestion before Phase 04 planning):**
- Watch source = `chokidar` (polling-fallback enabled by default; cross-platform).

**Ink + chokidar minimal setup:**
- Add `ink@^5.0.1` (React-based CLI rendering — modern build for Node 20+) and `react@^18.3.1` (peer dep for Ink) and `chokidar@^4.0.1` to `@swt-labs/cli` runtime deps.
- `packages/cli/src/commands/watch.ts` exports `watchHandler: CommandHandler`. On invocation:
  1. Resolve `cwd`, validate `.swt-planning/` exists (else USAGE_ERROR with the same message as `vibe init-redirect`).
  2. Build initial snapshot via `detectPhase({cwd})` from `@swt-labs/methodology`.
  3. `chokidar.watch(join(cwd, '.swt-planning'), {ignoreInitial: true})` — debounce change events (200ms) and re-run `detectPhase` on coalesced batches.
  4. `render(<Dashboard state={...} />)` — re-render on each fresh snapshot.
  5. SIGINT handler: `watcher.close()`, `instance.unmount()`, exit 0.

**Dashboard view model** (computed by `computeWatchState`):
- Project name + milestone slug (from STATE.md / ROADMAP.md aggregation in phase-detect output)
- Active phase: number + slug + state (needs_discussion / needs_plan_and_execute / needs_execute / needs_verification / etc.)
- Plan progress: N/M (summaries / plans)
- QA + UAT status flags
- Recent activity: last 3 commits touching the active phase dir (timestamps + first line)

**Pure function semantics:** `computeWatchState(snapshot, recentActivity)` is fully deterministic. Tests stage a snapshot literal + commit-log fixture and assert the view model shape. The TUI render layer is a thin wrapper over the pure model.

**Cross-platform notes:**
- chokidar polling fallback is auto-enabled on Windows network drives + WSL boundaries via `usePolling: false` default with `awaitWriteFinish` short timeout.
- macOS: native FSEvents.
- Linux: inotify. Both sub-second.

**Privacy note:** the watch dashboard does NOT emit telemetry events. F8 (Plan 04-03) ships HttpSender for the events that already fire from other commands; `swt watch` is read-only by design.
</context>
