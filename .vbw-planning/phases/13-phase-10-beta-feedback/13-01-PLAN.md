# Phase 13 / Plan 01: Opt-in telemetry implementation

## Why first

Telemetry needs to land before beta announcement (PLAN 13-02). Otherwise the first 10 beta users have nothing collected — friction reports become anecdotal. With telemetry opted-in (per user choice), even a single weekend of beta usage surfaces concrete usage patterns: which commands run, which phases stall, which remediation paths cycle.

## Layout

```

  index.ts                          # barrel
  client.ts                         # TelemetryClient
  sender.ts                         # Sender interface + NoopSender + TestSender
```

### T2 — TelemetryClient
Author `packages/telemetry/src/client.ts`:
```typescript
export class TelemetryClient {
  private buffer: TelemetryEvent[] = [];
  private flushTimer: NodeJS.Timeout | null = null;

  constructor(
    private readonly sender: Sender,
    private readonly enabled: boolean,
    private readonly anonymousId: string | null,
  ) {}

  send<E extends EventName>(name: E, properties: EventProperties[E]): void {
    if (!this.enabled || this.anonymousId === null) return;
    const sanitized = sanitize(name, properties);
    this.buffer.push({ name, properties: sanitized, anonymous_id: this.anonymousId, at: Date.now() });
    this.scheduleFlush();
  }

  async flush(): Promise<void> {
    if (this.buffer.length === 0) return;
    const batch = this.buffer.splice(0);
    try { await this.sender.send(batch); }
    catch { /* drop on send failure — no retries in v1.0 */ }
  }

  disable(): void { this.buffer = []; }

  private scheduleFlush(): void {
    if (this.flushTimer) return;
    this.flushTimer = setTimeout(() => { this.flushTimer = null; void this.flush(); }, 5000).unref();
  }
}
```

### T4 — first-run opt-in
Edit `packages/cli/src/commands/vibe.ts` to detect missing `telemetry` config block on first run. If `process.stdin.isTTY === true && !args.yolo`, prompt:

```
Help shape stop-wasting-tokens by enabling anonymous telemetry?

  - Tracks: which commands run, phase progress, UAT outcomes
  - NEVER tracks: file paths, repo names, user/project names, content
  - Anonymous ID generated locally; opt-out anytime via swt config

[y/N]
```

Default to `n`. Write `{enabled: <answer>, anonymous_id: <uuid>, opted_in_at: <iso-date>}` to config. Display `✓ Telemetry opt-in saved (you can change this anytime via 'swt config set telemetry.enabled true|false').`

Non-TTY / `--yolo` / CI: write `{enabled: false}` without prompting.
