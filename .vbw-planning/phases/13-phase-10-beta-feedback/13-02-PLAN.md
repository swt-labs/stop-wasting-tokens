# Phase 13 / Plan 02: Friction template + CODE_OF_CONDUCT + beta tester guide + announcement templates

## Why second

PLAN 13-01 ships the engineering layer (telemetry). PLAN 13-02 ships the *human* infrastructure beta testers interact with. Without templates and a code of conduct, the user can't credibly run a closed beta — Discord won't onboard without a CoC link, GitHub Issues without a friction template degenerate into noise, and announcements without copy-paste templates burn weekend hours on every channel.

## Tasks

### T1 — friction issue template
Author `.github/ISSUE_TEMPLATE/friction.md`:
```markdown
---
name: Beta friction report
about: Capture confusion, surprise, or "this should be smoother" from real usage
title: "[friction] "
labels: ["friction", "beta"]
assignees: []
---

## What were you trying to do?

<!-- One sentence: the goal. -->

## What happened?

<!-- What SWT actually did, with command output if relevant. -->

## What did you expect?

<!-- The mental model that didn't match. -->

## Severity

- [ ] Annoyance (worked, but felt clunky)
- [ ] Blocker (couldn't complete the task)
- [ ] Showstopper (data loss, corrupt state, can't recover)

## Environment

- SWT version: `swt --version`
- Node version: `node --version`
- OS: `<output of `uname -a` or Windows version>`
- Codex CLI version: `codex --version`

## Anything else?

<!-- Links, screenshots, related issues. -->
```

### T3 — GitHub Discussions templates
Create `.github/DISCUSSION_TEMPLATE/` directory.

`ideas.yml`:
```yaml
title: '[idea] '
labels: ['idea']
body:
  - type: textarea
    attributes:
      label: The idea
      description: One paragraph — what would change?
    validations:
      required: true
  - type: textarea
    attributes:
      label: Why now?
      description: What recent friction or insight motivated this?
  - type: textarea
    attributes:
      label: Tradeoffs you see
      description: What does this make worse?
```

`q-and-a.yml`:
```yaml
title: '[question] '
labels: ['question']
body:
  - type: textarea
    attributes:
      label: What are you trying to do?
      description: The end goal — not just the immediate command.
    validations:
      required: true
  - type: textarea
    attributes:
      label: What have you tried?
  - type: textarea
    attributes:
      label: Environment
      description: SWT/Node/OS versions
```

`show-and-tell.yml`:
```yaml
title: '[show] '
labels: ['show-and-tell']
body:
  - type: textarea
    attributes:
      label: What did you ship with SWT?
    validations:
      required: true
  - type: textarea
    attributes:
      label: Lessons learned
  - type: input
    attributes:
      label: Repo or screenshot link (optional)
```

import { describe, expect, it } from 'vitest';
import { existsSync, readFileSync } from 'node:fs';
import { join } from 'node:path';

const DOCS_ROOT = join(__dirname, '..');
const REPO_ROOT = join(DOCS_ROOT, '..');

describe('beta-feedback infrastructure', () => {
  it('beta-feedback.mdx exists and references the friction template', () => {
    const path = join(DOCS_ROOT, 'recipes', 'beta-feedback.mdx');
    expect(existsSync(path)).toBe(true);
    const content = readFileSync(path, 'utf8');
    expect(content).toContain('friction');
    expect(content).toContain('telemetry');
  });

  it('CODE_OF_CONDUCT.md exists at repo root', () => {
    expect(existsSync(join(REPO_ROOT, 'CODE_OF_CONDUCT.md'))).toBe(true);
  });

  it('friction issue template exists', () => {
    expect(existsSync(join(REPO_ROOT, '.github', 'ISSUE_TEMPLATE', 'friction.md'))).toBe(true);
  });

  it('all 3 GitHub Discussions templates exist', () => {
    const dir = join(REPO_ROOT, '.github', 'DISCUSSION_TEMPLATE');
    for (const name of ['ideas.yml', 'q-and-a.yml', 'show-and-tell.yml']) {
      expect(existsSync(join(dir, name)), name).toBe(true);
    }
  });

  it('all 4 announcement templates exist', () => {
    const dir = join(REPO_ROOT, '.vbw-planning', 'announcements');
    for (const name of ['discord-vbw-community.md', 'hacker-news-show.md', 'reddit-r-codex.md', 'twitter-x.md']) {
      expect(existsSync(join(dir, name)), name).toBe(true);
    }
  });
