## Tasks

### T1 — types/
Three discriminated string-literal types representing the resolution levers (effort, autonomy, verification tier). Each ships a `const` array of valid values for runtime checks.

### T2 — handoff/
Generic envelope wrapper: `{ from: AgentRole, to: AgentRole, kind: HandoffKind, payload: unknown, metadata: HandoffMetadata }` — payload narrowed by `kind`. Five concrete schemas (Scout, Architect, Lead, Dev, QA) — each is a Zod schema plus an inferred TypeScript type plus a `parse` helper that throws `HandoffError`.

### T3 — abstractions/
Four interfaces, no implementation. Pure type contracts. Each interface accompanied by short JSDoc covering responsibility and the lifecycle/event names it must support.

### T4 — config/
Single `Config` Zod schema covering: `effort`, `autonomy`, `verification_tier`, `model_profile`, `agent_max_turns` (record), `prefer_teams`, `auto_uat`, `planning_tracking`, `auto_push`. Defaults baked in via `.default()`. Inferred type `SwtConfig`.

### T5 — errors/
`SwtError` base class with `code`, `message`, optional `cause`. Subclasses: `ConfigError`, `HandoffError`, `PermissionDeniedError`, `MemoryError`, `BackendError`. Each subclass sets a literal `code` for narrowing.

### T6 — mock-driver.ts (test fixture)
Reference implementation satisfying all four abstractions. Used by tests to confirm the interfaces compose cleanly. No real I/O — in-memory only.

### T7 — Vitest suite
- `handoff.test.ts` — round-trip parse for each of the five schemas; rejection of malformed payloads; envelope wrapper enforces `from/to/kind` discriminator.
- `config.test.ts` — defaults applied; invalid values rejected; required fields enforced.
- `errors.test.ts` — `instanceof` narrowing works; `code` is preserved through serialisation.

### T8 — Commit + summary
`feat(core): add types, handoff schemas, four abstractions, config, error hierarchy`.
