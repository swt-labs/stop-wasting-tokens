<objective>
Make SWT installable via the Codex Plugin Marketplace by (1) moving the manifest to the documented path `.codex-plugin/plugin.json` (was `packages/cli/codex-plugin.json`), (2) restructuring fields to match the published Codex schema (`tags` → `keywords`, flat presentation fields → `interface` block, `author` string → object, drop undocumented `install`/`commands`), and (3) enforcing version-sync between the manifest and the npm package via a build-time test.

This closes F-03 (HIGH), F-13 (MEDIUM), and F-14 (LOW) from the v1.5.1 milestone scope.
</objective>
<context>
**Source research:** `.vbw-planning/research/20260507-081032-verify-codex-sdk-best-practices.md` — F-03, F-13, F-14.

**Documented Codex plugin manifest schema** (per `developers.openai.com/codex/plugins/build`):
- **Path:** `.codex-plugin/plugin.json` within the plugin root directory
- **Required top-level fields:** `name`, `version`, `description`
- **Optional top-level metadata:** `author` (object {name, email, url}), `homepage`, `repository`, `license`, `keywords`
- **Component pointers:** `skills` (string path), `mcpServers` (string path), `apps` (string path), `hooks` (string path)
- **`interface` block:** `displayName`, `shortDescription`, `longDescription`, `developerName`, `category`, `capabilities`, `websiteURL`, `privacyPolicyURL`, `termsOfServiceURL`, `defaultPrompt`, `brandColor`, `composerIcon`, `logo`, `screenshots`
- **No `$schema` documented as required** (Plan 01-02 DEV-2B already removed the placeholder; this plan keeps it absent)
- **No `install` or `commands` top-level fields documented** — Codex uses CLI auto-detect for npm packages and derives commands from skills, not from the manifest

**Per-role assignment for restructured manifest:**
- `name`: `"stop-wasting-tokens"` (unchanged)
- `version`: `"0.0.0"` (matches package.json — sync enforced by test)
- `description`: existing concise summary (unchanged)
- `author`: `{name: "Tiago Serôdio", url: "https://github.com/yidakee"}` (was bare string)
- `license`: `"MIT"` (unchanged)
- `homepage`: `"https://docs.stopwastingtokens.dev"` (unchanged)
- `repository`: `"https://github.com/swt-labs/stop-wasting-tokens"` (unchanged)
- `keywords`: `["methodology", "vibe-coding", "cli", "agents", "codex", "typescript", "monorepo"]` (was `tags`)
- `interface`: `{displayName: "stop-wasting-tokens (SWT)", category: "Development", screenshots: [...]}` (was flat top-level fields)
- **Removed:** `install`, `commands`, `tags`, `categories` (undocumented at top level)

**Test relocation:** the existing manifest test at `packages/cli/test/codex-plugin-manifest.test.ts` references `../codex-plugin.json` (i.e., `packages/cli/codex-plugin.json`). After the move, the manifest is at `.codex-plugin/plugin.json` (repo root). The test follows the manifest to a more central location at `test/codex-plugin-manifest.test.ts` (alongside `test/docs/drift.test.ts` from the v1.5 milestone). The vitest.config.ts include array already covers `test/**/*.test.ts`.

**Out of scope:**
- F-04 (subagent name/description fields) — already shipped in Phase 01.
- Build-time codegen for version sync at publish time — the test catches drift; an npm `prepublishOnly` hook is a separate machinery deferral.
- Validating the manifest against an actual Codex schema URL — Codex's `$schema` URL isn't documented (Plan 01-02 DEV-2B established this); the test asserts schema shape inline.
