<context>
**Decisions resolved 2026-05-07 (user via AskUserQuestion before Phase 03 planning):**
1. **Spawn mechanism** = `claude` CLI shell-out (parallels Phase 02's `codex exec --json`).
2. **API surface** for the spawn primitive: `claude -p <prompt> --output-format stream-json --model <model> --system-prompt <developer_instructions> --session-id <uuid>`. Tests stub the binary on `$PATH` (same approach Phase 02's CLI test used).

**`claude` CLI flags for `--print --output-format stream-json` mode (verified locally on 2026-05-07):**
- `-p / --print <prompt>` — non-interactive print mode
- `--output-format stream-json` — emits NDJSON envelopes (text chunks, tool_use, hook_events when `--include-hook-events`, final usage)
- `--model <model>` — model alias (`sonnet`, `opus`, `haiku`) or full ID
- `--system-prompt <prompt>` — system prompt for the session (used as developer_instructions)
- `--session-id <uuid>` — deterministic session id
- `--add-dir <directories...>` — additional access roots (used for cwd-aware tools)
- `--allowed-tools <tools>` — tool whitelist (used in higher-sandbox modes)
- `--include-hook-events` — surface lifecycle events in the output stream (used by Plan 03-02's HookHost)
- `--bare` — minimal mode (skip hooks, LSP, plugin sync) — used in tests to make output deterministic

**Stream-json envelope shape** (per Claude Code's documented schema):
- Text chunk: `{type: "assistant", message: {content: [{type: "text", text: "..."}]}}` or simpler `{text: "..."}`
- Handoff envelope: SWT's `{from, to, kind, payload, metadata: {created_at}}` shape, emitted as a tool result or final assistant message
- Usage chunk: `{type: "result", usage: {input_tokens, output_tokens}}`

**Agent profile persistence:** Claude Code reads agent definitions from `${CLAUDE_CONFIG_DIR:-~/.claude}/agents/${name}.json` (verified by `--agents <json>` flag and the `--agent <name>` lookup path). `installAgent(spec)` writes a JSON file with `{description, prompt, model, tools, sandbox_mode}` shape that matches Claude Code's agent JSON schema. `removeAgent` unlinks idempotently.

**Pre-existing v1.0 gap:** `packages/claude-code-driver/` has NO `tsconfig.json`. The package's typecheck fails with `TS5083: Cannot read file '...tsconfig.json'`. Plan 03-01 lands a tsconfig matching codex-driver's pattern (extends base, references core).
</context>
