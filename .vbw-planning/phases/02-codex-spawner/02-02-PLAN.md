<objective>
Round-trip token usage from Codex's `--json` stream all the way out to the AgentSpawner contract's SpawnResult.usage field. v1.0 left this as a placeholder (the type field exists in core but spawnCodex never populates it). F1 closes the loop: usage chunks → ParsedLine.usage → SpawnResult.usage → caller (methodology / telemetry).
</objective>
<context>
Codex CLI's `exec --json` emits NDJSON with mixed chunk types. Today the parser at `packages/codex-driver/src/spawn/parser.ts` recognises:
- generic `{ "text": "..." }` chunks
- structured handoff envelopes (validated by HandoffEnvelopeSchema)

It does NOT recognise `usage` chunks. Codex's documented usage chunk shape (per Codex CLI docs at v1.0 cut): `{ "type": "usage", "usage": { "input_tokens": N, "output_tokens": M } }` or similar.

**Caveat**: Codex's exact NDJSON schema is the source of truth here. If the actual chunk shape differs from the documented one, fixtures + parser must match the live schema. T2 captures a real fixture from a one-shot `codex exec --json` against a noop prompt to anchor the schema; T3's parser tests use the captured fixture.

**Schema in core** (already declared at packages/core/src/abstractions/AgentSpawner.ts):
```typescript
usage?: {
  readonly input_tokens: number;
  readonly output_tokens: number;
};
```

**Aggregation policy**: usage chunks may appear multiple times (intermediate + final). Default to last-write-wins (the final chunk is the canonical tally). Document this in code comments.
</context>
