# Phase 11 / Plan 03: Vale prose linting + CI integration

## Why third

Vale lands last because it lints content authored in Plans 11-01 + 11-02. Running Vale earlier would force us to fight false positives during content authoring; running it after means we configure rules against real prose. The CI integration is the success-criteria checkbox ("`vale` prose linting in CI passes"), so this plan closes the Phase 11 contract.

### T1 — Vale config + vocabulary
Author `apps/docs/.vale.ini`:
```ini
StylesPath = styles
MinAlertLevel = warning
Vocab = SWT

Packages = Microsoft, write-good

[*.{mdx,md}]
BasedOnStyles = Vale, Microsoft, write-good

# Reference docs are dense + technical — relax these
[reference/*.mdx]
Microsoft.Contractions = NO
Microsoft.We = NO
Microsoft.Adverbs = NO
write-good.Passive = NO
write-good.Weasel = NO
```

### T2 — bundle Vale style packages
Run `vale sync` once locally and commit `apps/docs/styles/Microsoft/` + `apps/docs/styles/write-good/`. Bundling avoids CI flakiness when the Vale package CDN is slow. Add `styles/Microsoft/` and `styles/write-good/` to git via explicit add — do NOT add to `.gitignore`.

If the package size is unwieldy (>5MB combined), fall back to a `vale sync` step in CI and a local bootstrap script `apps/docs/scripts/bootstrap-vale.sh`. Capture this decision in the PLAN summary.

### T3 — pre-commit hook script
Author `apps/docs/scripts/pre-commit-vale.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT_DIR="$(git rev-parse --show-toplevel)"
DOCS_DIR="$ROOT_DIR/apps/docs"

if ! command -v vale >/dev/null 2>&1; then
  echo "vale not installed — skipping prose lint" >&2
  exit 0
fi

CHANGED=$(git diff --cached --name-only --diff-filter=ACMR \
  | grep -E "^apps/docs/.*\.(mdx|md)$" || true)

[ -z "$CHANGED" ] && exit 0

cd "$DOCS_DIR"
vale --output=line $CHANGED
```
Mark executable. Document in `apps/docs/README.md`:
```bash
ln -s ../../apps/docs/scripts/pre-commit-vale.sh .git/hooks/pre-commit-vale
```

### T4 — initial lint pass + fix
Run `vale apps/docs/` against all content from PLANs 11-01 + 11-02. For each error: fix the prose. For each warning: either fix or add an inline disable with justification:
```mdx
<!-- vale Microsoft.Adverbs = NO -->
We _carefully_ designed the methodology to be ...
<!-- vale Microsoft.Adverbs = YES -->
```

Tally the resolutions in the SUMMARY: `N errors fixed, M warnings accepted with inline disables, K warnings remediated by rephrasing`.

### T5 — GitHub Actions workflow + vitest stub
Author `.github/workflows/vale.yml`:
```yaml
name: vale

on:
  pull_request:
    paths:
      - 'apps/docs/**'
      - '.github/workflows/vale.yml'

jobs:
  vale:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - name: Install Vale
        run: |
          curl -sL "https://github.com/errata-ai/vale/releases/download/v3.7.0/vale_3.7.0_Linux_64-bit.tar.gz" \
            | sudo tar -xzC /usr/local/bin vale
          vale --version
      - name: Lint docs
        run: vale --output=line apps/docs/
      - name: Build docs
        run: pnpm --filter @swt-labs/docs build
```

Author `apps/docs/test/vale.test.ts`:
```typescript
import { describe, expect, it } from 'vitest';
import { execSync } from 'node:child_process';

const isCI = process.env.CI === 'true';
const haveVale = (() => {
  try { execSync('vale --version', { stdio: 'ignore' }); return true; }
  catch { return false; }
})();

describe.skipIf(!haveVale && !isCI)('vale prose lint', () => {
  it('returns zero error-severity violations', () => {
    const out = execSync('vale --output=JSON apps/docs/', { encoding: 'utf8' });
    const json = JSON.parse(out);
    const errors: any[] = [];
    for (const file of Object.values(json) as any[][]) {
      for (const v of file) if (v.Severity === 'error') errors.push(v);
    }
    expect(errors, JSON.stringify(errors, null, 2)).toEqual([]);
  });
});
```
