<objective>
Extend the SWT hook event taxonomy from 6 generic events to 12, adding the six lifecycle events the methodology layer needs (pre_archive, post_phase, pre_phase, post_uat_fail, pre_qa, post_qa). Add a `hooks` config block so users can wire scripts to these events. Ship narrowing helpers per the abstractions audit. Provide one sample shell script per new event.
</objective>
<context>
**Decisions resolved 2026-05-07 (user via AskUserQuestion before Phase 05 planning):**
- Hook config shape = nested `hooks` object in ConfigSchema with one optional sub-block per event.

**Extended HookEvent union** in `packages/core/src/abstractions/HookHost.ts`:
```typescript
export type HookEvent =
  // v1.0 generic events
  | 'session_start'
  | 'user_prompt_submit'

  | 'post_tool_use'

  | 'stop'
  // v1.5 SDLC lifecycle events

```

**Narrowing helpers** (one per event):
```typescript
export function isPreToolUseEvent(
  ctx: HookContext,
): ctx is HookContext & { event: 'pre_tool_use' } {
  return ctx.event === 'pre_tool_use';
}
// ... 11 more
```

**Sample scripts** in `templates/hooks/`:
- Each script starts with `#!/usr/bin/env bash`
- Has a comment block describing when it fires + what env vars it receives (e.g., `$SWT_PHASE`, `$SWT_PLAN`)
- Body is a `printf` to stdout that demonstrates the event firing — harmless on dry-run, no destructive side effects
- Mode `0755` (executable)

**post_archive semantics unchanged:** the v1.0 `post_archive` hook (currently invoked from `scripts/post-archive-hook.sh`) is NOT renamed or repurposed. Plan 05-03 adds NEW events; existing semantics are preserved.

```bash
#!/usr/bin/env bash
# pre_archive — fires before `swt vibe --archive` moves a milestone from
# .swt-planning/ to .swt-planning/milestones/{slug}/.
#
# Env vars:
#   $SWT_MILESTONE_SLUG  — the slug being archived
#   $SWT_PHASE_COUNT     — number of phases in the milestone
#
# Use this hook to: stage release notes, run final lint sweeps, post a
# Slack notification, etc. Exit non-zero to abort the archive.

printf 'pre_archive: archiving milestone %%s with %%s phases\\n' "$SWT_MILESTONE_SLUG" "$SWT_PHASE_COUNT"
```
