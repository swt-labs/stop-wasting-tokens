<objective>
Replace the v1.0 hand-authored `docs/reference/{cli,config,artifacts}.mdx` files with codegen output. The generator is a tsx-runnable `scripts/docs-gen.ts` that imports the published source surfaces directly (ConfigSchema for config.mdx; the CLI command registry for cli.mdx; @swt-labs/artifacts Zod schemas for artifacts.mdx) and emits MDX with the same overall shape, including hand-authored "When to override" prose merged from `docs/reference/_prose/`.
</objective>
<context>
**Decisions resolved 2026-05-07 (user via AskUserQuestion before Phase 05 planning):**
1. **Codegen mechanism** = hand-rolled tsx script. No new runtime or dev deps; ~150-300 LOC.
2. **Prose merge** = side-files in `docs/reference/_prose/{cli,config,artifacts}.mdx` with marker comments (`<!-- PROSE:override-config-effort -->`) that the generator inlines at output time.

**Generator surface** (scripts/docs-gen.ts):
- `generateCliMdx(): string` — reads buildRegistry from @swt-labs/cli, emits a markdown table with command name + usage + description + (optional) inline override prose.
- `generateConfigMdx(): string` — reads ConfigSchema from @swt-labs/core, walks the top-level keys via `ConfigSchema.shape`, emits a section per key with `**Type:**`, `**Default:**`, plus inlined override prose from `_prose/config.mdx`.
- `generateArtifactsMdx(): string` — reads each published Zod schema in @swt-labs/artifacts (PlanFrontmatterSchema, SummaryFrontmatterSchema, VerificationFrontmatterSchema, etc.) and emits a section per artifact type with the field listing.
- `main()` — runs all three generators and writes to `docs/reference/{cli,config,artifacts}.mdx`.

**Prose marker shape:** the side-files use sectioned markdown:
```
<!-- PROSE:override-config-effort -->
`turbo` for prototypes, `thorough` for production. See [effort levels](/concepts/effort-levels).
<!-- /PROSE -->
```
The generator scans for `<!-- PROSE:<id> -->` ... `<!-- /PROSE -->` blocks and inlines the content at matching marker locations in the generated output.

**npm script:** add `"docs:gen": "tsx scripts/docs-gen.ts"` to root `package.json`.

**AUTO-GENERATED header:** the generated MDX starts with a comment `<!-- AUTO-GENERATED by scripts/docs-gen.ts. Do not edit by hand. Edit prose in docs/reference/_prose/. -->` — this replaces the v1.0 `AUTO-DERIVE-CANDIDATE` marker.

**Out of scope for Plan 05-01:**
- Build-time drift check (lands in Plan 05-02).
- Generation for other docs surfaces (handoff schemas, hook events) — only the three roadmap-listed files in scope.
- Migration of all existing prose into the side-files; Plan 05-01 captures the "When to override" prose only for the keys that already have it in v1.0.
</context>
