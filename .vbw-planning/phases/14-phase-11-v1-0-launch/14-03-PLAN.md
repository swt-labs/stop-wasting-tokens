# Phase 14 / Plan 03: VBW deprecation notice + demo video script + launch checklist

## Why third

PLANs 14-01 and 14-02 land the artifacts the user will reference at launch (release notes, blog post, security review). PLAN 14-03 lands the launch-day operating manual: VBW deprecation copy, demo video script, and a single checklist that consolidates every user-side action from Phases 11–14 into one ordered walkthrough. Without this consolidation, the user would have to re-read 14 SUMMARY.md files to figure out what to do on launch day.

```
.vbw-planning/announcements/
  vbw-deprecation-notice.md     # NEW
  demo-video-script.md          # NEW
LAUNCH-CHECKLIST.md             # NEW (repo root)
docs/v1-5-roadmap/index.mdx     # MOVE deployment item to launch-day
packages/core/test/
  launch-checklist.test.ts      # NEW
```

# Demo video script — v1.0 launch

Target length: 6 minutes (5–8 acceptable). Pacing: deliberate, no fast-cuts.
Tone: matter-of-fact, no hype. Show the runtime, don't sell it.

## 0:00 — Cold open (15s)

[B-roll: terminal showing a long Codex session with repeated re-explaining of context]

## 0:15 — Problem framing (45s)

[B-roll: split screen — chat history on left, .swt-planning/ tree on right]

> "stop-wasting-tokens structures the work so token spend goes toward _new_ progress. Phases on disk. Plans on disk. Verification on disk. Codex reads the contract; you read the diff; nothing evaporates between sessions."

## 1:00 — Install + init (60s)

[Live terminal: npm install -g @swt-labs/cli, then mkdir demo-todo && cd demo-todo && swt init]

> "One install. One init. Two prompts: planning_tracking, auto_push. Defaults are safe — manual + never. Existing repo? It detects brownfield and runs codebase mapping."

## 2:00 — Plan + execute Phase 1 (120s)

[Live: swt vibe → bootstrap dialog → scope into 3 phases → swt vibe → plan + execute Phase 1]

> "Plan + execute on the first phase. Lead spawns, writes 01-01-PLAN.md. Dev executes each task. QA verifies row-by-row against the must-haves. The runtime knows where you are at every step."

## 4:00 — Verification + UAT (90s)

[Live: swt vibe → auto-routes to UAT → walk through 3 checkpoints, fail one, watch remediation chain]

> "UAT is interactive. You walk through scenarios. One fails. The runtime routes to remediation: research → plan → execute → re-verify. Bounded round caps. Recurrence tracking if a fix doesn't stick."

## 5:30 — Archive + reset (30s)

[Live: swt vibe → audit → milestone tagged → STATE.md reset]

> "All phases pass. swt vibe runs the 7-point audit. Milestone archives to .swt-planning/milestones/01-todo-cli/. Git tag created. State resets to project-level. Next milestone starts clean."

## 6:00 — CTA (30s)

[Static: install command + docs URL]

> "npm install -g @swt-labs/cli. docs.stopwastingtokens.dev. Closed beta open this week. Try it. Tell me what's broken."

## End slate (~5s)

URL + GitHub icon + Discord icon.

## Production notes

- Record at 1080p minimum, 60fps preferred for terminal cuts
- Use a clean monospace font (JetBrains Mono / Fira Code), light theme on dark background
- Mic check before record — no AC hum, no keyboard clack
- Two takes minimum per section; pick the one that lands the timing
- Subtitles required (auto-gen via YouTube + manual review)

# Launch checklist — v1.0

Use this checklist on launch day. Items are ordered: pre-flight first, then publish, then announce, then monitor.

## Pre-flight (do once, days before launch)

- [ ] Configure `NPM_TOKEN` secret in GitHub Actions (Settings → Secrets and variables → Actions)
- [ ] Verify `release.yml` workflow runs cleanly on a draft tag (e.g., `v0.0.0-test`, then delete)
- [ ] Confirm `pnpm install --frozen-lockfile && pnpm typecheck && pnpm lint && pnpm test` is green
- [ ] Review `SECURITY-REVIEW-v1.0.md` — every PASS row reflects current code, every FOLLOW-UP is acknowledged
- [ ] Replace placeholder URLs across the repo:
  - [ ] `docs.stopwastingtokens.dev` → real Mintlify hosting URL (after DNS + Mintlify setup)
  - [ ] `discord.gg/swt-labs-beta` → real Discord invite
  - [ ] `docs.codex.example` → real Codex Plugin Marketplace schema URL
  - [ ] `conduct@stopwastingtokens.dev` → real CoC contact email
- [ ] Set up Mintlify hosting + DNS CNAME for `docs.stopwastingtokens.dev` (Phase 11 deferral)
- [ ] Create Discord server with Code of Conduct pinned (Phase 13 deferral)

## npm publish (the actual ship)

- [ ] `scripts/bump-version.sh 1.0.0 --dry-run` — review
- [ ] `scripts/bump-version.sh 1.0.0` — apply
- [ ] `git diff` — sanity check
- [ ] `git add -A && git commit -m "chore(release): v1.0.0"`
- [ ] `git tag -a v1.0.0 -m "v1.0.0 — first stable release"`
- [ ] `git push origin main v1.0.0` — triggers `release.yml`
- [ ] Watch Actions: `release.yml` publishes 7 packages; `install-smoke.yml` validates the install (6-cell matrix)
- [ ] Verify on npm: `npm view @swt-labs/cli` shows v1.0.0 + provenance attestation

## Marketplace submission

- [ ] Submit `packages/cli/codex-plugin.json` + `packages/cli/MARKETPLACE.md` to the Codex Plugin Marketplace per its submission process
- [ ] Add real `$schema` URL to `codex-plugin.json` once Codex publishes the marketplace manifest schema

## Docs deploy

import { describe, expect, it } from 'vitest';
import { existsSync, readFileSync } from 'node:fs';
import { join } from 'node:path';

const REPO_ROOT = join(__dirname, '..', '..', '..');
const PATH = join(REPO_ROOT, 'LAUNCH-CHECKLIST.md');

describe('LAUNCH-CHECKLIST.md', () => {
  it('exists at repo root', () => {
    expect(existsSync(PATH)).toBe(true);
  });

  it('has the 7 canonical sections', () => {
    const content = readFileSync(PATH, 'utf8');
    const sections = [
      '## Pre-flight',
      '## npm publish',
      '## Marketplace submission',
      '## Docs deploy',
      '## VBW deprecation',
      '## Announcements',
      '## Post-launch',
    ];
    for (const section of sections) {
      expect(content, `missing: ${section}`).toContain(section);
    }
  });

  it('references the engineering→ship handoff items', () => {
    const content = readFileSync(PATH, 'utf8');
    for (const ref of [
      'NPM_TOKEN',
      'bump-version.sh',
      'codex-plugin.json',
      'discord',
      'docs.stopwastingtokens.dev',
      'release.yml',
    ]) {
      expect(content, `missing reference: ${ref}`).toMatch(new RegExp(ref, 'i'));
    }
  });
});
