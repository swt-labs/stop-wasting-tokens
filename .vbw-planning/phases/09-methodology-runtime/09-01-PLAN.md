# Phase 9 / Plan 01: phase-detect TS port

Source: `vibe-better-with-claude-code-vbw-main/scripts/phase-detect.sh` (1604 lines bash) → `packages/methodology/src/state/phase-detect.ts` (TypeScript).

## Why first

`swt vibe`'s entire routing layer depends on this function. Until it exists in TypeScript, every state-driven mode (Verify, UAT Remediation, Re-verify, Plan + Execute, Execute, Archive, Milestone UAT Recovery, Discuss) cannot be implemented. Nothing else in Phase 9 can land without it.

## Tasks

### T1 — types.ts
Mirror every key the bash script emits. Use string-literal unions for enum-ish fields so consumers can switch exhaustively. Required keys (matching VBW exactly, lowercase snake_case):
`jq_available`, `planning_dir_exists`, `project_exists`, `phases_dir`, `has_shipped_milestones`, `needs_milestone_rename`, `phase_count`, `next_phase`, `next_phase_slug`, `next_phase_state`, `next_phase_plans`, `next_phase_summaries`, `has_unverified_phases`, `first_unverified_phase`, `first_unverified_slug`, `first_qa_attention_phase`, `first_qa_attention_slug`, `qa_attention_status`, `qa_attention_reason`, `qa_status`, `qa_reason`, `qa_round`, `uat_issues_phase`, `uat_issues_slug`, `uat_issues_major_or_higher`, `uat_issues_phases`, `uat_issues_count`, `uat_file`, `uat_round_count`, `misnamed_plans`, `milestone_uat_issues`, `milestone_uat_phase`, `milestone_uat_slug`, `milestone_uat_major_or_higher`, `milestone_uat_phase_dir`, `milestone_uat_count`, `milestone_uat_phase_dirs`, `execution_state`, `phase_detect_complete` plus the `config_*` mirror fields.

### T2 — scan-phases.ts
Walk `<planningDir>/phases/`, ordered numerically by leading two-digit prefix. For each `<NN>-<slug>/` directory, return a typed `PhaseSnapshot` with the artefacts present (`{plan_count, summary_count, has_context, has_research, verification, uat, remediation_round_count, qa_remediation_state, uat_remediation_state}`).

### T3 — classify-phase.ts
Given a `PhaseSnapshot`, return one of:
- `needs_discussion` — no `<NN>-CONTEXT.md` and `require_phase_discussion=true`
- `needs_plan_and_execute` — has CONTEXT (or discussion not required) but no PLAN.md
- `needs_execute` — has PLAN(s) but missing SUMMARYs
- `needs_verification` — fully built, no terminal `<NN>-UAT.md`
- `needs_uat_remediation` — has `<NN>-UAT.md` with `status: issues_found`
- `needs_qa_remediation` — has FAIL/PARTIAL VERIFICATION or active `qa-remediation-state`
- `needs_reverification` — UAT remediation state is `done` and round-scoped UAT is missing/stale
- `complete` — UAT terminal-pass, PLAN-coverage-complete, no active remediation

Plus the QA freshness staleness check fed by `qa-freshness.ts`.

### T4 — milestone-uat.ts
Scan `<planningDir>/milestones/<slug>/phases/<…>/` for any `*-UAT.md` with `status: issues_found` AND no `.remediated` marker. Aggregate per-affected-phase context.

### T5 — qa-freshness.ts
Read `verified_at_commit` from VERIFICATION frontmatter, run `git rev-parse HEAD` (or fallback when not a repo), report whether product code changed since QA passed. Also fall back to `working_tree_changed` when `git status --porcelain` is non-empty for product paths.

### T6 — encode.ts
`toKeyValueLines(result): string` — flatten the typed result back into the `key=value` shape the bash script emits, sorted in the original order. Used by the CLI debug command and by any script-driven consumer that still expects the bash format. This keeps `swt detect-phase --bash-format` compatible with anything written for VBW.

### T7 — phase-detect.ts
Compose T2–T6 plus config loading. Single entry point. Returns the typed result.

### T8 — CLI integration
Add `swt detect-phase` to `packages/cli/src/commands/` (real impl, not stub). Optional `--json` (default) and `--bash-format` flags.

### T9 — Vitest fixtures
Build twelve `.swt-planning/` fixtures under `test/state/fixtures/`. Each fixture is a tiny tree (directories + a few markdown files) representing one of the documented states. Tests assert the exact `next_phase_state` plus a handful of supporting keys.

### T10 — Commit + summary
`feat(methodology): port VBW phase-detect to TypeScript with full state coverage`.
