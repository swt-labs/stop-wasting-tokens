# Phase 6 Plan: Commands

Source: `.vbw-planning/research/source-plan.md` § G, Phase 5.

## Tasks

### T1 — exit-codes.ts
Three constants: SUCCESS=0, USAGE_ERROR=1, NOT_IMPLEMENTED=2. Imported everywhere instead of magic numbers.

### T2 — argv.ts
Wraps `node:util` `parseArgs` with SWT-specific global flags (--effort, --skip-qa, --yolo, --plan, --version, --help) and normalised positional arguments.

### T3 — router.ts
Command registry. Each command exposes `{ name, description, handler }`. Router dispatches the verb to its handler; unknown verbs print help + suggestion + exit 1.

### T4 — Real commands
- `swt help` — list verbs + descriptions, dump global flags.
- `swt --version` / `swt version` — print package version + git rev (if available).
- `swt config show` / `get <key>` / `set <key> <value>` — read/write `.swt-planning/config.json` via `parseConfig` from core.
- `swt status` — read `.swt-planning/STATE.md` (fail gracefully if missing) and pretty-print the current phase, milestone, recent activity.
- `swt doctor` — check Node version, codex on PATH (via codex-driver detectCodexVersion), git presence, pnpm presence, .swt-planning dir presence.

### T5 — Stub commands
Single factory: `stubCommand(name, roadmapPointer): CommandHandler`. Each stub prints an explanatory message and a pointer to the roadmap section, returns exit 2.

### T6 — Tests
- argv: positional + flag + unknown flag rejection
- router: known + unknown verb behaviour
- help: lists every registered command
- version: includes the package.json version
- doctor: shape of the report (no real shell calls in tests — mock detectCodexVersion)
- status: reads a fake STATE.md and prints it

### T7 — Update packages/cli/src/index.ts
Replace the smoke main() with the new main from main.ts and re-export it.

### T8 — Commit + summary
`feat(cli): argv parser + router + real cmds (help/version/status/config/doctor) + stubs`.
