# Phase 12 / Plan 03: Codex Plugin Marketplace metadata + install verification

## Why third

Plans 12-01 and 12-02 ship the engineering layer; Plan 12-03 closes the user-discovery loop. Without a marketplace listing, Codex users have to find SWT manually. With it, they install via the marketplace UI — same path as any other Codex plugin. Plan 12-03 also adds the post-publish smoke test so the first failed install doesn't reach users.

```
  MARKETPLACE.md                    # listing copy (separate from npm README)
```

### T3 — verify-install.sh
Author `scripts/verify-install.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

VERSION="${1:?usage: verify-install.sh <expected-version>}"

# Confirm binary is on PATH
if ! command -v swt >/dev/null 2>&1; then
  echo "✗ swt not on PATH after install" >&2
  exit 1
fi

# Version match
INSTALLED=$(swt --version 2>/dev/null | head -1)
if [[ ! "$INSTALLED" =~ "$VERSION" ]]; then
  echo "✗ Version mismatch: expected $VERSION, got $INSTALLED" >&2
  exit 1
fi

# Init scaffolds in a tmp dir
TMP_DIR=$(mktemp -d)
pushd "$TMP_DIR" >/dev/null
if ! swt init --yes 2>/dev/null; then
  echo "✗ swt init failed in fresh directory" >&2
  popd >/dev/null
  rm -rf "$TMP_DIR"
  exit 1
fi
[ -f .swt-planning/PROJECT.md ] || { echo "✗ swt init did not create PROJECT.md" >&2; exit 1; }
popd >/dev/null
rm -rf "$TMP_DIR"

# detect-phase round-trips
swt detect-phase --json --planning-dir /tmp/nonexistent 2>/dev/null \
  | grep -q "init_required" || { echo "✗ detect-phase did not return init_required for missing dir" >&2; exit 1; }

echo "✓ Install verification passed for v${VERSION}"
```
Mark executable.

### T4 — install-smoke workflow
Author `.github/workflows/install-smoke.yml`:
```yaml
name: install-smoke

on:
  workflow_run:
    workflows: ["Release"]
    types: [completed]
    branches: [main]
  push:
    tags: ['v*']

jobs:
  smoke:
    if: ${{ github.event_name == 'push' || github.event.workflow_run.conclusion == 'success' }}
    runs-on: ${{ matrix.os }}
    timeout-minutes: 10
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest]
        manager: [npm, pnpm, bun]

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install via ${{ matrix.manager }}
        run: |
          case "${{ matrix.manager }}" in
            npm) npm install -g @swt-labs/cli ;;
            pnpm) npm install -g pnpm && pnpm add -g @swt-labs/cli ;;
            bun) curl -fsSL https://bun.sh/install | bash && export PATH="$HOME/.bun/bin:$PATH" && bun add -g @swt-labs/cli ;;
          esac

      - name: Verify install
        run: bash scripts/verify-install.sh ${{ github.ref_name }}
        env:
          PATH: ${{ env.PATH }}:${{ runner.tool_cache }}/bin
```
Notes: the `bun add -g` path needs PATH munging so subsequent steps see the bun bin directory. Document in SUMMARY as a known matrix quirk.

```typescript
import { describe, expect, it } from 'vitest';
import { readFileSync } from 'node:fs';
import { join } from 'node:path';

const PKG_DIR = join(__dirname, '..');
const manifest = JSON.parse(readFileSync(join(PKG_DIR, 'codex-plugin.json'), 'utf8'));
const cliPkg = JSON.parse(readFileSync(join(PKG_DIR, 'package.json'), 'utf8'));

describe('codex-plugin.json', () => {
  it('declares required fields', () => {
    expect(manifest.name).toBe('stop-wasting-tokens');
    expect(manifest.install?.npm).toBe('@swt-labs/cli');
    expect(manifest.install?.command).toBe('swt');
    expect(manifest.tags).toEqual(expect.arrayContaining(['methodology', 'cli', 'codex']));
    expect(Array.isArray(manifest.commands)).toBe(true);
    expect(manifest.commands.length).toBeGreaterThanOrEqual(4);
  });

  it('version matches package.json', () => {
    expect(manifest.version).toBe(cliPkg.version);
  });
});
```
