# Phase 12 / Plan 01: npm publish wiring + provenance + version sync

## Why first

Without publishable manifests, none of Phase 12's deliverables matter — `swt update` (PLAN 12-02) needs to query a real npm registry record, and the Codex Plugin Marketplace listing (PLAN 12-03) requires a real install path. Plan 01 unblocks both.

## Layout

```
packages/{core,cli,codex-driver,methodology,artifacts,verification,telemetry}/
  package.json             # private: false + publishConfig
.changeset/
  config.json              # ignore: [] + linked: [[@swt-labs/*]]
.github/workflows/
  release.yml              # NPM_TOKEN + id-token:write + --provenance
scripts/
  bump-version.sh          # lockstep version bump, no auto-push
package.json               # publishConfig + version sync
README.md                  # install section
packages/cli/test/
  publish-config.test.ts   # drift detection
```

## Tasks

### T1 — flip packages to publishable
For each `packages/*/package.json`:
- Remove `"private": true` (or set to `false`)
- Add:
  ```json
  "publishConfig": {
    "access": "public",
    "provenance": true
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/swt-labs/stop-wasting-tokens.git",
    "directory": "packages/<NAME>"
  },
  "bugs": {
    "url": "https://github.com/swt-labs/stop-wasting-tokens/issues"
  },
  "homepage": "https://github.com/swt-labs/stop-wasting-tokens#readme",
  "license": "MIT",
  "author": "Tiago Serôdio (@yidakee)"
  ```

### T2 — changeset config
Update `.changeset/config.json`:
- Empty `ignore: []` (was: 7 entries)
- Replace `linked: []` with `linked: [["@swt-labs/core","@swt-labs/cli","@swt-labs/codex-driver","@swt-labs/methodology","@swt-labs/artifacts","@swt-labs/verification","@swt-labs/telemetry"]]` (lockstep)
- Keep `access: "public"` and `commit: false`

### T3 — release.yml provenance
Update `.github/workflows/release.yml`:
- Confirm `permissions.id-token: write` (already present)
- Add `NPM_TOKEN: ${{ secrets.NPM_TOKEN }}` to the changesets/action `env`
- changesets/action will pick up `publishConfig.provenance: true` from each package automatically — no separate `--provenance` flag needed in scripts

### T4 — bump-version.sh
Author `scripts/bump-version.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

NEW_VERSION="${1:?usage: bump-version.sh <semver>}"
DRY_RUN="${2:-}"

ROOT="$(git rev-parse --show-toplevel)"
PACKAGES=(core cli codex-driver methodology artifacts verification telemetry)

if [ "$DRY_RUN" = "--dry-run" ]; then
  echo "Would bump root + ${#PACKAGES[@]} packages to v${NEW_VERSION}"
  for p in "${PACKAGES[@]}"; do
    echo "  - packages/$p/package.json"
  done
  echo "  - package.json (root)"
  exit 0
fi

echo "Bumping all packages to v${NEW_VERSION}..."
for p in "${PACKAGES[@]}"; do
  manifest="$ROOT/packages/$p/package.json"
  node -e "const m=JSON.parse(require('fs').readFileSync('$manifest'));m.version='$NEW_VERSION';require('fs').writeFileSync('$manifest',JSON.stringify(m,null,2)+'\n')"
done
node -e "const m=JSON.parse(require('fs').readFileSync('$ROOT/package.json'));m.version='$NEW_VERSION';require('fs').writeFileSync('$ROOT/package.json',JSON.stringify(m,null,2)+'\n')"

echo "✓ All 8 manifests at v${NEW_VERSION}"
echo "Next: review with 'git diff', then 'git tag v${NEW_VERSION} && git push origin v${NEW_VERSION}'"
```
Mark executable. Add a per-CLAUDE.md guard at the top: `# CLAUDE.md rule: do not run without explicit user request.`

Add publishConfig validation vitest at `packages/cli/test/publish-config.test.ts`:
```typescript
import { describe, expect, it } from 'vitest';
import { readFileSync } from 'node:fs';
import { join } from 'node:path';
import { glob } from 'node:fs/promises';

const REPO_ROOT = join(__dirname, '..', '..', '..');
const PACKAGES = ['core','cli','codex-driver','methodology','artifacts','verification','telemetry'];

describe('publishConfig parity', () => {
  for (const pkg of PACKAGES) {
    it(`@swt-labs/${pkg}: publishConfig + provenance`, () => {
      const manifest = JSON.parse(
        readFileSync(join(REPO_ROOT, 'packages', pkg, 'package.json'), 'utf8')
      );
      expect(manifest.publishConfig?.access).toBe('public');
      expect(manifest.publishConfig?.provenance).toBe(true);
      expect(manifest.private).not.toBe(true);
      expect(manifest.repository?.url).toContain('swt-labs/stop-wasting-tokens');
      expect(manifest.license).toBe('MIT');
    });
  }
});
```
