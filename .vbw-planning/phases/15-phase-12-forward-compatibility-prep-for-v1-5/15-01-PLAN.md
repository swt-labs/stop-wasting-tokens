# Phase 15 / Plan 01: Core abstractions audit + Claude Code + Ollama driver stubs

## Why first

Phase 15 closes v1.0 by setting up v1.5. The abstractions audit identifies any Codex-specific leakage that would block a Claude Code / Ollama driver from implementing the same interface — better to find leakage now than mid-implementation in v1.5. The stub packages reserve the npm names + establish the contract surface.

import type { AgentSpawner, AgentSpec, AgentSession } from '@swt-labs/core';

const NOT_IMPLEMENTED =
  '@swt-labs/claude-code-driver: not implemented in v1.0; lands in v1.5 — track docs/v1-5-roadmap';

export class ClaudeCodeAgentSpawner implements AgentSpawner {
  spawn(_spec: AgentSpec): Promise<AgentSession> {
    throw new Error(NOT_IMPLEMENTED);
  }
}

export const PACKAGE_NAME = '@swt-labs/claude-code-driver';
export const VERSION = '0.0.0';
```

# @swt-labs/claude-code-driver

The full implementation lands in v1.5: 12-event hook taxonomy, Agent Teams, isolation modes (REQ-V2-02). Track progress at [`docs/v1-5-roadmap`](https://docs.stopwastingtokens.dev/v1-5-roadmap/index).

Until then, use [`@swt-labs/codex-driver`](../codex-driver/) for live execution.
```
