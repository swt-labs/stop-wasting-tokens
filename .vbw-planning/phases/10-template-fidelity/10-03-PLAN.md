# Phase 10 / Plan 03: REMEDIATION + DEBUG-SESSION + CONTEXT schemas

## Why third (and last typed-shape plan)

PLAN 10-01 covered PLAN/SUMMARY. PLAN 10-02 covered VERIFICATION/UAT/RESEARCH. PLAN 10-03 closes the contract by adding the four remediation artifacts (REMEDIATION-PLAN/RESEARCH/SUMMARY + DEBUG-SESSION) and the two CONTEXT artifacts (per-phase + milestone). After this lands, every VBW artifact kind has a typed Zod schema in `@swt-labs/artifacts/schemas/`. PLAN 10-04 (template strings) is then optional polish — the schemas ARE the contract.

## Layout

```
packages/artifacts/src/schemas/remediation-plan.ts
packages/artifacts/src/schemas/remediation-summary.ts
packages/artifacts/src/schemas/remediation-research.ts
packages/artifacts/src/schemas/debug-session.ts
packages/artifacts/src/schemas/context.ts            # phase + milestone
packages/artifacts/src/schemas/index.ts              # extend the barrel
packages/artifacts/test/schemas/remediation.test.ts  # all three remediation schemas
packages/artifacts/test/schemas/debug-session.test.ts
packages/artifacts/test/schemas/context.test.ts
```

## Tasks

### T1 — RemediationPlanSchema
Frontmatter: phase, round (`/^\d{2}$/`), title, tasks_total, fail_classifications (array of `{id, type: 'code-fix'|'plan-amendment'|'process-exception', rationale, source_plan?}`), known_issues_input (string[] of JSON), known_issue_resolutions (string[] of JSON). Read/write helpers.

### T2 — RemediationSummarySchema
Frontmatter: phase, round, title, status, completed, tasks_completed/total, commit_hashes, files_modified, deviations, known_issue_outcomes (string[] of JSON). Read/write helpers.

### T3 — RemediationResearchSchema
Frontmatter: phase, round, title, gathered, sources_consulted, files_referenced, findings_summary, live_validation_required. Read/write helpers.

### T4 — DebugSessionSchema
Frontmatter: session_id, started, agent (`'debugger' | 'qa' | 'dev'`), phase?, plan?, status (`'open' | 'resolved' | 'abandoned'`), summary (one-sentence). Body sections: `## Investigation`, `## Findings`, `## Resolution`. Read/write helpers.

### T5 — PhaseContextSchema (per-phase)
Frontmatter: phase, slug, name, goal, requirements (string[]), success_criteria (string[]), pre_seeded (bool). Body sections: `## Goal`, `## Notes`, `## Decisions`, `## Deferred Ideas`. Read/write helpers.

### T6 — MilestoneContextSchema
Frontmatter: gathered, calibration (`'builder' | 'architect'`), milestone_name. Body sections: `## Scope Boundary`, `## Decomposition Decisions` (with `### Scope Coverage` subsection), `## Requirement Mapping`, `## Key Decisions`, `## Deferred Ideas`. Read/write helpers.

### T7 — Vitest
- `remediation.test.ts`: round-trip RemediationPlan/Summary/Research with realistic fixtures.
- `debug-session.test.ts`: parse a session with all three body sections; reject malformed agent.
- `context.test.ts`: parse + write phase context (with pre_seeded=true); parse + write milestone context.

### T8 — Commit + summary
`feat(artifacts): REMEDIATION + DEBUG-SESSION + CONTEXT schemas (Phase 10 / PLAN 03)`.
