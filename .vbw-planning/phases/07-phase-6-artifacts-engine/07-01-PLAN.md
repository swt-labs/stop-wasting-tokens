## Tasks

### T1 — frontmatter.ts
Tiny `---`-fenced frontmatter parser. Body and a flat YAML scalar / array map. Sufficient for SWT's emitted shapes; intentionally not a full YAML parser.

### T2 — atomic-write.ts
Write to `<path>.tmp-<pid>-<random>`, fsync, rename over `<path>`. Avoids torn writes when SWT and a user editor both touch STATE.md.

### T3 — Zod schemas
Loose schemas matching what SWT writes today. Strict-but-permissive: every required key validated, unknown keys preserved on round-trip via a `passthrough` mode where helpful.

### T4 — State updater
`readState(path)` returns the parsed shape; `updateState(path, mutator)` applies a mutation function and writes atomically.

### T5 — Roadmap editor
- `addPhase(roadmap, name, opts)` appends.
- `insertPhase(roadmap, position, name, opts)` shifts later phases up by 1 and renumbers.
- `removePhase(roadmap, position)` shifts later phases down by 1 and renumbers.
All three return the new ROADMAP body string + the migrations needed (renames, etc.) so callers can sync the filesystem.

### T6 — Phase layout
`createPhaseDir(planningDir, position, slug)` makes the directory and writes a CONTEXT.md skeleton. Idempotent.

### T7 — Milestone archive
`archiveMilestone(planningDir, slug)` moves ROADMAP.md, the current STATE.md, and `phases/` into `milestones/<slug>/`, then writes a fresh root STATE.md preserving project-level sections (Todos, Decisions, Blockers).

### T8 — Tests
Schema round-trip, frontmatter parse/format, atomic-write durability under concurrent calls, roadmap add/insert/remove correctness, milestone archive moves the right files.

### T9 — Commit + summary
`feat(artifacts): schemas + frontmatter IO + STATE/ROADMAP editors + milestone archive`.
